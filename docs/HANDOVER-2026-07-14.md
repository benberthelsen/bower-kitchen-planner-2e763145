## Working constraints (IMPORTANT — read before editing)

1. **The sandbox file mount lags/truncates.** After editing a file, the Linux
   sandbox (bash/tsc) often reads a TRUNCATED copy — e.g. tsc reports a parse
   error at line 194 of a 193-line file. These are false. The host files (what
   the `Read`/`Edit` tools see) are the source of truth. Verify with `Read`, not
   bash `cat`/`tsc`, after editing. Rewriting a whole large file via bash heredoc
   from host-read content is the escape hatch when mount truncation blocks work.
2. **Do NOT commit from the agent session** — Ben commits from his side (the
   sandbox's file view lags, so agent commits can capture truncated files).
3. **Do NOT deploy edge functions from the sandbox** (no Supabase CLI/network).
   Ben deploys. See deploy command below.
4. **Supabase writes are gated** by an auto-approval classifier — don't retry-spam
   denied writes.
5. **Public bundle must never carry raw supplier costs.** Material identity comes
   from the public `planner-materials.json`; COST is authoritative from the DB
   (`material_pricing`, RLS authenticated-only).

## Key architecture pointers

- **Pricing engine**: `src/lib/pricing/` (`generateQuoteBOM`). Single source of
  truth — planner room quote, configurator Parts List (`useCabinetBOM`), and
  wizard band all use it. Commercial layer ×1.35 + GST; per-cabinet SELL =
  cost-share of room total (`useTradeRoomPricing.ts`).
- **3D**: planner room view = `PlannerScene.tsx`; homeowner/configurator preview =
  `UnifiedScene.tsx` + `CabinetAssembler.tsx` (shared parts in
  `components/3d/cabinet-parts/`). Corner cabinets need the arm-depth/secondWidth/
  square-footprint mapping (`src/lib/cornerDefaults.ts`) — used by RoomPlanner,
  Preview3D, and ProductThumbnail3D.
- **Catalog**: `useCatalog.ts` (DB `microvellum_products` + static templates).
- **Website→planner handoff**: `planner_handoffs` table (anon INSERT only);
  payload = styleTags + materials + dimensions + notes.
